# Soccer Stat

SPA приложение на React (TypeScript + Vite) для просмотра футбольных лиг, команд и календаря матчей. Разработано как тестовое задание в SimbirSoft.

🔗 **[Демо на GitHub Pages](https://kinzhalproject.github.io/SimbirSoft_solution/)**

## 🌟 Особенности реализации

* **UI/UX:** Максимально легкая реализация: написан собственный адаптивный UI (Flexbox, Grid) с кастомной пагинацией и DatePicker, без использования тяжелых UI-библиотек (Ant Design был полностью удален). Бандл оптимизирован (~100 КБ gzip).
* **Архитектура API:** Реализовано In-Memory кэширование (на 5 минут) с In-Flight дедупликацией параллельных запросов для минимизации лишних сетевых вызовов. Ошибка `429 Too Many Requests` (которая является ограничением бесплатного тарифа football-data.org) сведена к минимуму. Отключен `React.StrictMode` для предотвращения дублирования запросов в dev-режиме (особенно на первый рендер).
* **Мобильная адаптивность:** Поддержка всех основных брейкпоинтов экранов. Адаптивные карточки и таблицы с горизонтальным скроллом. Сохранен pixel-perfect дизайн из ТЗ.
* **Настройка окружения:** Соблюдены правила чистой работы с git: `.env` с ключами исключен из коммитов, добавлены строгие проверки `ESLint` и форматирование `Prettier`.

## 🚀 Установка и запуск

1. Клонируйте репозиторий:
   ```bash
   git clone <URL>
   cd soccer-stat
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Создайте файл `.env` в корне проекта (рядом с `package.json`) и добавьте свой API-ключ от `football-data.org`:
   ```env
   VITE_FOOTBALL_API_KEY=ваш_api_ключ_здесь
   ```

4. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

**⚠️ Важно о проверке через GitHub Pages:**
Сам провайдер API `football-data.org` по умолчанию блокирует все CORS запросы из браузера, отправленные откуда-либо кроме `http://localhost`. Бесплатные публичные прокси (corsproxy.io, allorigins) выдают 500 ошибку из-за их бана по IP серверами API. Поэтому настроены честные прямые API вызовы. 
**Для оценки работы на GitHub Pages необходимо временно установить CORS Unblocker плагин**, либо запустить проект локально через `npm run dev` (где Vite Proxy решает эту проблему на 100%).
